
# תכנית: מערכת פרופיל משתמש עם צפייה אדמינית

## סקירה כללית

מערכת פרופיל משתמש מקיפה שמאפשרת:
1. **לכל משתמש** - לראות ולערוך את הפרופיל האישי שלו
2. **לאדמין** - לצפות בפרופיל של כל משתמש אחר ולראות את כל המידע שלו
3. **למאמנים** - לצפות בפרופיל של הלקוחות שלהם בלבד

```text
מבנה הנתיבים:
┌─────────────────────────────────────────────────────────────────┐
│  /profile                    → הפרופיל האישי שלי                │
│  /profile/:userId            → צפייה בפרופיל של משתמש אחר       │
│  /community/profile/:userId  → פרופיל ציבורי בקהילה             │
│  /panel/users/:userId        → צפייה אדמינית מלאה               │
└─────────────────────────────────────────────────────────────────┘
```

## מה יופיע בפרופיל

### פרופיל בסיסי (ציבורי)
| שדה | מקור | נגיש ל |
|-----|------|--------|
| שם מלא | profiles.full_name | כולם |
| תמונה | community_members.avatar_url | כולם |
| ביו | profiles.bio | כולם |
| רמה ונקודות | community_members | כולם |
| תאריך הצטרפות | profiles.created_at | כולם |

### מידע מורחב (אדמין/מאמן)
| שדה | מקור | נגיש ל |
|-----|------|--------|
| אימייל | auth.users (via edge function) | אדמין |
| טלפון | user_sensitive_data | המשתמש עצמו בלבד |
| תפקידים | user_roles | אדמין |
| רכישות | purchases | אדמין, מאמן (שלו) |
| מנויים | user_subscriptions | אדמין |
| התקדמות לאונצ'פד | launchpad_progress | אדמין, מאמן |
| סיכום Aurora | launchpad_summaries | אדמין, מאמן |
| סשנים | hypnosis_sessions | אדמין, מאמן |
| צ'קליסטים | aurora_checklists | אדמין, מאמן |
| תוכנית חיים | aurora_focus_plans | אדמין, מאמן |

## הרשאות ו-RLS

### בדיקת הרשאות בקוד:
```text
צפייה בפרופיל:
├── משתמש רגיל
│   ├── פרופיל עצמי: מלא (חוץ מאימייל)
│   └── פרופיל אחר: רק מידע ציבורי (קהילה)
├── מאמן
│   ├── פרופיל עצמי: מלא
│   ├── לקוחות שלו: Aurora + רכישות
│   └── אחרים: ציבורי בלבד
└── אדמין
    └── כל משתמש: הכל (כולל אימייל, תפקידים)
```

### שינויי RLS נדרשים:
- **profiles**: כבר מאפשר קריאה לכולם
- **launchpad_summaries**: צריך policy לאדמין
- **purchases**: צריך policy למאמן לראות רכישות של הלקוחות שלו
- **hypnosis_sessions**: policy לאדמין/מאמן

## אבטחה

**חשוב!** הנתונים הרגישים (טלפון) נשמרים ב-`user_sensitive_data` עם RLS שמאפשר גישה רק למשתמש עצמו - זה נשאר כך.

האימייל יתקבל דרך Edge Function `get-user-data` שכבר קיימת ומשמשת את Users.tsx.

אדמין יוכל לראות:
- פרטי פרופיל
- אימייל (דרך edge function)
- תפקידים
- רכישות ומנויים
- נתוני Aurora
- היסטוריית פעילות

אדמין **לא** יוכל לראות:
- טלפון (מוגן ב-user_sensitive_data)
- סיסמאות (מעולם לא חשופות)
