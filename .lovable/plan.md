
# MindOS — Lovable Exit & Migration Audit

This is an audit + action plan only. No code, schema, or behavior changes are proposed in this phase.

---

## 1. Project Structure

- **Shape**: Single Vite + React app at the repo root. `apps/evolve` and `backend/openclaw` exist as scaffolds (see `management/DIRECTORY_ALIGNMENT.md`) but the **real source still lives at root `src/` and `supabase/`**. Treat the scaffold as aspirational, not active.
- **Package manager**: npm/bun compatible — `package.json` only, no workspaces declared. Vercel uses `npm run build`.
- **Build**: Vite 5 + SWC + Tailwind 3 + `vite-plugin-pwa`. TS 5.8.
- **Entry points**: `index.html` → `src/main.tsx` → `src/App.tsx` (≈73 `<Route>` declarations).
- **External services**: Supabase (DB/Auth/Storage/Edge Functions), Lovable AI Gateway + OpenRouter (via `supabase/functions/_shared/aiGateway.ts`), ElevenLabs (TTS + transcribe), Stripe (checkout/portal/coach checkout), Web3Auth (Sapphire Devnet), Vercel (hosting + `api/` Node functions).
- **Env vars (frontend)**: `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`, `VITE_SUPABASE_PUBLISHABLE_DEFAULT_KEY`, `VITE_SUPABASE_PROJECT_ID`, plus optional `VITE_ENABLE_*` feature flags and `VITE_AGENT_API_BASE_URL` (see `.env.example`).
- **Env vars (backend / edge)**: `LOVABLE_API_KEY`, `OPENROUTER_API_KEY` (optional fallback path), `ELEVENLABS_API_KEY`, `STRIPE_SECRET_KEY`, `RESEND_API_KEY` (newsletter/email), `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_URL`, `SUPABASE_ANON_KEY`. Confirm with `secrets--fetch_secrets` at export time.
- **Deployment assumptions**: `vercel.json` builds `api/**/*.ts` via `@vercel/node`, outputs SPA from `dist`, and rewrites all non-`/api` paths to `index.html`. Edge Functions assumed deployed automatically by Lovable (`.github/workflows/deploy-functions.yml` exists for off-Lovable parity).
- **Lovable-specific surfaces**:
  - `src/integrations/lovable/index.ts` + `@lovable.dev/cloud-auth-js` package — Lovable-managed OAuth.
  - `src/integrations/supabase/client.ts` and `src/integrations/supabase/types.ts` — auto-regenerated by Lovable; need to be hand-maintained externally.
  - `.env` is auto-managed; outside Lovable you maintain it manually.
  - `LOVABLE_API_KEY` (used by `_shared/aiGateway.ts`) must be replaced or kept as OpenRouter fallback path.

## 2. Backend / Supabase

- **Migrations**: 257 migration files in `supabase/migrations/`. Migration history is the canonical schema source — fully replayable on a fresh project via `supabase db push`.
- **Edge functions**: 73 functions under `supabase/functions/`. Notable clusters: AION (`aion-brain`, `aion-orchestrator`, `aion-capabilities`, `aurora-chat`, `aurora-*`), generation (`generate-100day-strategy`, `generate-coach-plan`, `generate-business-plan`, `generate-curriculum`, `generate-hypnosis-script`, `generate-orb-narrative`, `generate-milestone-journey`, etc.), commerce (`create-checkout-session`, `create-coach-checkout`, `customer-portal`, `check-subscription`, `check-coach-subscription`, `fm-approve-claim`), admin (`admin-grant-purchase`, `admin-refresh-users`), media (`elevenlabs-tts`, `elevenlabs-transcribe`, `cache-hypnosis-audio`, `get-audio-by-token`, `get-video-by-token`), web3 (`web3-wallet`, `web3auth-exchange`).
- **Storage buckets**: declared in migrations (`rg "storage.buckets" supabase/migrations` returns multiple). Must `supabase storage ls` and dump objects for any non-regenerable assets (hypnosis audio cache, branding uploads, avatars, generated images).
- **Auth**: Email/password + Google OAuth + Lovable-managed OAuth bridge. JWT verification in functions handled in code (most functions ship `verify_jwt = false` per Lovable defaults).
- **RLS / RPC / triggers**: Defined throughout migrations. Key functions to confirm survive: `award_unified_xp`, `has_role`, `update_updated_at_column`, plus pillar/quest triggers. No raw client SQL is invoked.
- **What must be dumped vs recreated**:
  - **Recreatable from migrations**: schema, RLS, functions, triggers, enums, buckets (definitions).
  - **Requires production dump**: `auth.users`, `profiles`, `user_roles`, AION memory tables, journals, `action_items`, `ai_generations`, `dna_*`, conversation/message history, subscription records, storage objects.
- **Required secrets at new project**: `SUPABASE_SERVICE_ROLE_KEY`, `LOVABLE_API_KEY` *(or replace with `OPENROUTER_API_KEY`)*, `ELEVENLABS_API_KEY`, `STRIPE_SECRET_KEY`, `RESEND_API_KEY`, plus any Web3Auth client/verifier keys if kept active.

## 3. Web3 / Wallet

- **Packages**: `@web3auth/modal` (v10). No `wagmi`, `viem`, or `ethers`. Pure Web3Auth modal + a custom edge bridge (`web3-wallet`, `web3auth-exchange`).
- **Files**: `src/lib/web3auth.ts`, `src/lib/web3authConfig.ts`, `src/providers/Web3AuthProviderWrapper.tsx`, `src/components/auth/Web3AuthModalBridge.tsx`, `src/components/auth/Web3AuthLoginButton.tsx`, `src/components/web3/SoulAvatarMintWizard.tsx`.
- **Chain config**: Sapphire **Devnet** (per memory). No production contract addresses identified in repo — confirm before disabling.
- **Risks**: `buffer`/`process` polyfills already configured in `vite.config.ts`. Web3Auth modal can collide with PWA service worker if domain changes — re-register OAuth verifier under new domain.
- **Migration recommendation**: keep code, gate behind feature flag `VITE_ENABLE_NFT_INVENTORY=false` (already in `.env.example`). Remove from default bundle via lazy import if cost matters.

## 4. AI / LLM / Voice

- **Gateway**: `supabase/functions/_shared/aiGateway.ts` auto-routes between OpenRouter (when `OPENROUTER_API_KEY` set) and Lovable AI Gateway (when `LOVABLE_API_KEY` set). Includes a `MODEL_MAP` translating Lovable model IDs (`google/gemini-3-flash-preview`, `openai/gpt-5*`) to OpenRouter equivalents. **This is your single migration seam — keep it intact.**
- **Edge functions calling AI**: `aurora-chat`, `aion-brain`, `aion-orchestrator`, `aurora-analyze`, `aurora-recalibrate`, `aurora-proactive`, `domain-assess`, `consciousness-assess`, all `generate-*` functions, `analyze-*`, `negotiate-plan`, `course-orchestrator`, `career-wizard`, `daily-priority-rebalance`, `daily-blog-generator`.
- **TTS / Voice**: `elevenlabs-tts` + `elevenlabs-transcribe` + `cache-hypnosis-audio` + `services/ttsSpeak.ts` + `services/voiceCapture.ts`. Hash-based cache (per memory `cached-tts-system-v2`).
- **Stream handling**: `_shared/sanitizeStream.ts` + `aurora-chat/sanitizeStream.ts` (with `sanitizeStream_test.ts`). Enforces `hideReasoning` / final-answer-only on reasoning models.
- **Prompts**: live in edge functions (server-side only — already correct).
- **Migration note**: `OPENROUTER_API_KEY` already a first-class path. To exit Lovable AI: set `OPENROUTER_API_KEY`, leave `LOVABLE_API_KEY` unset → gateway flips automatically.

## 8. Minimal External Setup (Run Book)

```text
1. Export source: zip repo from Lovable (or git clone the auto-mirrored repo).
2. cp .env.example .env  → fill VITE_SUPABASE_URL, VITE_SUPABASE_PUBLISHABLE_KEY,
   VITE_SUPABASE_PROJECT_ID.
3. npm install
4. npm run dev                              # localhost:8080
5. supabase login && supabase link --project-ref <ref>
6. supabase db push                         # replays 257 migrations
7. supabase functions deploy --project-ref <ref>   # deploys all 73
8. Set edge secrets:
     supabase secrets set OPENROUTER_API_KEY=...   # AI
     supabase secrets set ELEVENLABS_API_KEY=...   # voice (optional)
     supabase secrets set STRIPE_SECRET_KEY=...    # payments (optional)
     supabase secrets set RESEND_API_KEY=...       # email (optional)
9. Configure Auth: enable Email + Google in Supabase dashboard,
   set Site URL + redirect URLs for new domain.
10. Vercel: import repo, framework = Vite, output = dist, env vars from .env.
11. Stripe webhook → repoint to new functions URL.
12. Web3Auth (optional): register new verifier for new domain, leave behind feature flag.
```

## 10. Slimming Strategy

**Phase A — Migration Stabilization (zero behavior change)**
- Freeze `src/integrations/supabase/{client,types}.ts` snapshot.
- Replace `LOVABLE_API_KEY` path with `OPENROUTER_API_KEY` in production secrets.
- Decouple `@lovable.dev/cloud-auth-js` from critical flows; keep as optional path.
- Verify all 73 edge functions deploy via the GH workflow against a staging Supabase project.
- Confirm storage bucket export + DB dump restore on staging.

**Phase B — Dead/Legacy Code Removal**
- Delete `src/_legacy/`.
- Archive `src/hallway/` and `flows/pillarSpecs/` into `_archive/`.
- Drop unused root pages (`Business`, `BusinessDashboard`, `BusinessJourney` if not routed) after route audit.
- Remove unused npm deps confirmed by `depcheck` (likely candidates: `html2canvas`, `jspdf`, `openai`, `embla-carousel-*`, `recharts`).
- Consolidate stream sanitizers into one `_shared/sanitizeStream.ts`.

**Phase C — Ontology Cleanup & Lightweight Runtime**
- Single canonical orb path (`OrbView` only) — strip stragglers per `orb-pure-renderer-standard`.
- Single canvas (`unified-orb-stage-v4`) — audit for accidental second canvases.
- Collapse `selfworld/`, `world/`, `worlds/` into the canonical `worlds/` ontology.
- Move to monorepo target (`apps/evolve`, `backend/supabase`) per `management/DIRECTORY_ALIGNMENT.md`.
- Trim unused realm/pillar legacy data; keep AION core (`aion-chat`, `aion-brain`, `aion-orchestrator`, `aurora-*`) untouched.
